  • HTTP

    HTTP stands for HyperText Transfer Protocol. It is a protocol that allows your computer to connect to servers on the web and returns any requests such as html pages.

  • Client

    A client is any computer or device that is making the requests to the server.

  • Server

    A server is a program or device that fulfills the requests of the client.

  • Web Browser

    A web browser is a program on your computer/client which allows you to navigate the web and displays wesbites.

  • Website

    A website is a set of web pages all under the same domain name with related content to each other.

  • Web page

    A web page is a HyperText document which displays content on a website.

  • JavaScript

    JavaScript is a programming language based around objects and used within web pages.

  • HTML

    HyperText Markup Language. HTML is a markup language that structures the text and layout of a web page.

  • CSS

    CSS stands for Cascading Style Sheets. CSS is a language that defines the look and the style of a web page. It is usually attached to an html file.

  • URL

    The address or link to a web page.

  • IDE / text editor

    A program that allows you to read and write various programming languages.

  • Command Line / Terminal

    An interface where you can execute commands such as making, removing, and copying files or directories.

  • BASH

    BASH is a shell that works similar to a Command Line/Terminal.

  • Operating System

    The program a computer runs on that manages all tasks and other programs.

  • Algorithm

    An Algorithm is the instructions

  • Frontend vs Backend Development

    Frontend is focused on the looks of the website while backend focuses on it's inner workings.

  • Programming vs Coding

    Programming is the planning and logic behind the project while coding is the execution of the tasks needed to complete.

  • Git vs Github

    Git is a version control system used to store all the stages of your project. Github is the website used to view and manage your projects and the projects of other people.

  • Github terms: push, pull, fork, clone, repository

    push is to publish your changes to your repository. pull copies over any new changes in the remote repository to your local storage so they are the same. fork copies someone else's repository into a new repository for yourself that doesn't affect the original. clone copies the original repository to your local system and any changes made are reflected in the original. A repository stores all the files and folders for a project and the revision history.
